当你遇到报错时，最好的方式不是“反复点”，而是 **按顺序排查**。这份清单会带你一步步把问题缩小到可解决的范围。

---

## 0. 先做 30 秒快速体检（强烈推荐）
请按顺序做这四件事（很多问题到这里就解决了）：
1. 回到主页面点击一次 **刷新余额**
2. 检查你选择的 **模式** 是否正确（普通/跨链/质押）
3. 检查收款地址是否粘贴完整（不要多空格/换行）
4. 普通/快速模式下，对收款地址点击一次 **盾牌验证**

## 1. 构造交易常见错误与处理

### 1.1 “找零地址缺失 / 缺少找零地址”
原因：系统需要把“多出来的部分”找零回你自己的地址，但找零地址为空。

处理：
1. 打开“高级选项”
2. 为对应币种选择找零地址
3. 跨链模式必须选择 **PGC 找零地址**

如果你明明看到下拉框里有值但仍提示缺失：
- 轻点一下下拉框重新选择一次
- 或刷新页面再试（这类问题通常是 UI 状态不同步导致的）

### 1.2 “找零地址不存在 / 找零地址错误”
原因：找零地址不在你的钱包地址列表中，或币种类型不匹配。

处理：
- 选择一个确实存在于左侧地址列表的找零地址
- 确保币种对应（PGC 找零就选 PGC 地址）

### 1.3 “公钥格式错误 / 公钥缺失”
原因：普通/快速模式构造输出需要收款方公钥。

处理：
1. 收款地址输入后点击盾牌验证（自动补全公钥）
2. 若补全失败，确认该地址是否已在链上存在/可查询

### 1.4 “地址格式错误”
常见原因：
- 地址粘贴时带了空格或换行
- 跨链模式要求 `0x...`，你填的是普通地址
- 胶囊地址未先验证解析

处理：
- 删除空格，重新粘贴
- 确认模式与地址格式匹配
- 胶囊地址先完成验证/解析

### 1.5 “余额不足 / Gas 不足”
原因可能很多，但处理路径基本固定：
1. 换一个余额更高的来源地址
2. 或多选几个来源地址（普通/快速支持）
3. 若提示 Gas 不足：在高级选项里适当增加 Extra Gas 或选更多来源地址
4. 如果你刚发过交易：等待上一笔确认后再试

### 1.6 “跨链限制”
跨链模式常见触发点：
- 收款人超过 1 个
- 选择了多个来源地址
- 金额不是整数
- 没选 PGC 找零地址
- 收款地址不是 0x 开头

处理：逐项对照修正即可。

## 2. 提交后不更新 / 状态异常

### 2.1 状态一直 Pending
建议顺序：
1. 等待 1～3 分钟
2. 刷新余额/交易记录
3. 查看历史记录是否出现新的状态

如果一直不变：
- 可能是网络/节点不可用
- 或本地正在降级为轮询，更新会慢一些

### 2.2 反复提交失败
请避免连续点击造成资源锁定：
- 等待 10～30 秒后再试
- 刷新页面让锁定状态恢复

## 3. 联调环境常见问题（开发/测试同学）
如果你在本地联调（前端 + 后端）：
- 前端默认 Gateway：`http://localhost:3001`
- 先确认后端服务已启动并能访问 `/health`
- 打开浏览器开发者工具（Console/Network）查看请求是否报错

如果需要更系统的本地启动步骤，请阅读《开发者快速开始》。

---

## 4. 如何提供“有效信息”给开发同学（提高解决速度）
当你需要把问题交给开发同学时，建议附上：
- 发生问题时的模式（普通/跨链/质押）
- 收款币种、金额、收款地址格式（可打码中间部分）
- 你选择了哪些来源地址（可打码）
- 页面报错提示原文
- 控制台 Console 的错误日志（复制文本比截图更好）

你不需要把所有细节都搞清楚——把信息记录下来，我们就能更快定位并修复问题。
